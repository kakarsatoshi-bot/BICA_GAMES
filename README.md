# MOS QUEST 〜ワード・エクセル ぼうけんの書〜

MOS（Word / Excel）の授業向け・試験対策RPGゲームです。
生徒はモンスターとのバトル（選択式クイズ）でレベルを上げ、成績をきそいます。

**すべて無料で運用できます**（サーバー代・管理費 0円）。

---

## ゲームの内容

| 要素 | 内容 |
|---|---|
| クエスト | エクセルの塔（6クエスト60問）＋ワードの城（6クエスト60問）＝計120問。MOS 365の公式出題範囲（ブック/文書管理・印刷含む）をカバー |
| かしこい出題 | 各クエストは問題プールから毎回8問を「未挑戦→復習期限→苦手」の優先順で自動選出。周回するほど弱点だけが出題される |
| バトル | 4択問題に正解するとモンスターを倒せる。まちがえるとハートが減る |
| ボス | 各クエストの最後3問はボス戦。全クエスト制覇で「まおうの城」が解放 |
| もぎけんてい | 本試験と同じ1000点満点・合格ライン700点の20問模試（Excel/Word別）。分野別の苦手診断つきで、最高スコアは先生のシートにも記録される |
| 経験値 / レベル / ランク | 正解でEXP獲得。連続正解ボーナスあり。レベルに応じて「みならい冒険者→…→でんせつの勇者」と称号が上がる |
| ふっかつのほこら（復習） | **忘却曲線（間隔反復）** にもとづき、忘れかけたタイミングの問題と、まちがえた問題だけを自動で出題 |
| きおくの地図 | 問題ごとの習熟度（記憶の定着度）を色で見える化 |
| 成績の送信・共有 | 生徒の成績は先生のスプレッドシートに自動送信。クラス全体のランキングは先生がClassroom等で共有（詳しくは後述） |
| バトル背景 | クエストごとに森・洞くつ・城・荒野・塔…と背景が変わる（すべてコード描画のオリジナルドット絵） |
| ご意見箱 | 画面右下の📮からいつでも感想・要望・問題の間違い報告を投稿できる。スプレッドシートの「ご意見箱」シートに届く |
| なまえの変更 | 「ぼうけんのしょ」画面からいつでも名前・クラスを変更できる（成績は引き継がれる） |
| モンスター図鑑 | 遭遇したモンスターに、Excel/Wordの「あるある」ネタの一言バイオが解放される（「ぼうけんのしょ」から） |
| 激レアモンスター | ごく低確率で「コピペマスター」が出現するお楽しみ要素。ハートは減らない安全なボーナス戦 |
| 試験日カウントダウン | MOSの試験日を設定すると、メニュー画面に王さま・お姫さま・まおうが残り日数に応じて一言くちを出す |

### 記憶定着のしくみ（ライトナー方式）

問題ごとに「記憶ボックス0〜5」を管理しています。

- 正解するとボックスが1つ上がり、次の復習は先になる（1日後→3日後→7日後→14日後→30日後）
- まちがえるとボックス0に戻り、すぐ復習対象になる

エビングハウスの忘却曲線に合わせ「忘れかけたころに再出題」されるため、
定期的に遊ぶだけで記憶が定着します。「ふっかつのほこら」に赤い数字が出ていたら復習どきです。

## MOS単語帳（日本語がまだ苦手な生徒向け）

`vocab` フォルダに、MOS QUESTとは別の単独アプリとして「MOS単語帳」を同梱しています。
日本語を勉強中の外国人生徒が、MOS受験に必要な日本語の専門用語（セル・書式設定・差し込み印刷など）を
**英語訳＋ふりがな**付きで覚えられる、シンプルなフラッシュカード＋クイズアプリです。

- 対応カテゴリー：きょうつう操作／Excel専用／Word専用／復習（まだ覚えていない語だけ）
- フラッシュカードモード：カードをタップでめくって「おぼえた／もう一度」を記録
- クイズモード：4択クイズ（日本語→英語、または「英語→日本語」に切り替え可）
- 学習の記録はブラウザのlocalStorageに保存（成績送信はしません。個人情報のやり取りは一切ありません）
- 単語データは `vocab/js/words.js` にあり、MOS QUESTの問題と同じ書き方（配列に足すだけ）で追加・編集できます

MOS QUESTのタイトル画面から「📘 MOS単語帳」リンクで開けるほか、`vocab/index.html` を直接開いても遊べます。
公開方法はMOS QUESTと同じで、`vocab` フォルダごとGitHub Pages等にアップロードすればそのまま動きます。

> 💡 `vocab/js/*.js` や `vocab/css/*.css` を更新したときは、ブラウザのキャッシュで古いファイルが
> 表示され続けるのを防ぐため、`vocab/index.html` の `?v=2` の数字を1つ増やしてください
> （例: `js/app.js?v=2` → `js/app.js?v=3`）。増やさないと、生徒の端末に前のバージョンが
> キャッシュされたままになり、新機能が反映されないことがあります。

## 公開のしかた（どれか1つでOK）

ゲームは `game` フォルダだけで完結する静的サイトです。ビルドやサーバーは不要です。

> ⚠️ **注意**: この bica-materials リポジトリ自体は公開しないでください（授業の解答ファイルが入っているため）。
> 公開するのは `game` フォルダの中身だけです。

### 方法A: GitHub Pages（おすすめ・無料）

1. GitHubで新しい**公開（Public）リポジトリ**を作る（例: `mos-quest`）
2. `game` フォルダの中身（index.html, css, js）をそのリポジトリにアップロードする
   （GitHubの画面で「Add file → Upload files」にドラッグ＆ドロップでOK）
3. リポジトリの Settings → Pages → Branch を `main` / `(root)` にして Save
4. 数分後に表示されるURL（`https://ユーザー名.github.io/mos-quest/`）を生徒に配る

### 方法B: Netlify Drop（アカウント登録すれば無料・ドラッグ＆ドロップだけ）

1. https://app.netlify.com/drop を開く
2. `game` フォルダをそのままドラッグ＆ドロップ
3. 表示されたURLを生徒に配る

### 方法C: 校内PCで直接開く

`game/index.html` をダブルクリックするだけでも遊べます（成績の送信以外の全機能が動きます）。

## 成績の送信・共有の設定（やらなくてもゲームは動きます）

先生からの進捗確認を有効にする手順です。約10分で終わります。

1. **Googleスプレッドシートを新規作成**（名前は「MOS QUEST 成績」など自由）
2. メニューの **拡張機能 → Apps Script** を開く
3. 最初から書いてあるコードを全部消して、このリポジトリの
   **`gas/mos_quest_leaderboard.gs`** の中身をぜんぶ貼り付けて保存
4. 右上の **デプロイ → 新しいデプロイ** をクリック
   - 種類の選択（歯車マーク）→ **ウェブアプリ**
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
   - デプロイ → アクセスを承認（自分のGoogleアカウントを選んで許可）
5. 表示された **ウェブアプリのURL**（`https://script.google.com/macros/s/...../exec`）をコピー
6. Apps Scriptの「プロジェクトの設定」（左メニューの歯車マーク）→「スクリプト プロパティ」で
   **`API_KEY`** という名前のプロパティを追加し、値には**パスワードのようなランダムな文字列**を設定する
   （例: 大文字小文字と数字を混ぜた20文字程度。誰にも教えないこと）
7. `game/js/config.js` を開いて、次の3か所を書きかえて公開し直す

```javascript
SYNC_URL: "ここにコピーしたURLを貼る",
API_KEY: "手順6で決めたのと まったく同じ文字列",
CLASSES: ["1年A組", "1年B組"],   // 実際のクラス名に書きかえる
```

これで、生徒がゲームをプレイすると自動的にスプレッドシートへ成績が送信されるようになります。
**このAPIは送信専用（write-only）です。** 生徒全員の成績を一覧で読み取れる機能はあえて用意していません（理由は次の「セキュリティについて」参照）。

### ランキング（成績一覧）の共有方法

ゲームには「クラス全員のランキングを表示する画面」はありません。かわりに、
先生がスプレッドシートを**組織内限定・閲覧のみ**で共有し、Google Classroom等の
学内システム経由で生徒に配布する運用にしています（Google純正の権限管理を使うため、
学校の外には一切見えません）。

1. 「Players」シートを開く
2. 右上の「共有」ボタン → 「制限付き」を「〇〇（学校のドメイン名）」に変更
3. 権限は「閲覧者」のままにする（生徒が編集できないように）
4. 発行されたリンクを、Google Classroomの「授業」または「ストリーム」に投稿する

見やすく整えたい場合は、シートをコピーして生徒向けの表示用シート（名前・クラス・レベル・EXPだけを
`QUERY`関数などで抜き出したもの）を作り、そちらを共有するのがおすすめです。

### 先生の確認方法

**スプレッドシート（元の「Players」シート）を開くだけ**で、生徒全員のレベル・EXP・正解率・
クエスト達成数・最終プレイ日時が1人1行で見られます（自動更新）。

### ご意見箱

生徒がゲーム内の📮ボタンから送った感想・要望は、同じスプレッドシートに自動作成される
**「ご意見箱」シート**に届きます（受信日時・なまえ・クラス・内容・送った画面）。
こちらも送信専用で、生徒が他の生徒の投稿を読むことはできません。

> 💡 コードを修正して再デプロイするときは「デプロイを管理 → 編集 → 新バージョン」を選んでください（URLが変わりません）。

## 試験日カウントダウンの設定（やらなくてもゲームは動きます）

MOSの試験日を設定すると、メニュー画面に王さま・お姫さま・まおうが残り日数に応じて
一言くちを出すようになります（GAS側の設定は不要、ゲーム内だけで完結します）。

MOSの試験日はインターネットから自動取得することはできません（学校ごとに個別で
手配される日程のため、外部サイトを機械的に読み取っても正しい日付とは限りません）。
そのため、先生が把握している試験日を `game/js/config.js` の `EXAM_DATES` に
**わかっている分だけリストで**入力する方式にしています。

```javascript
EXAM_DATES: ["2026-09-04", "2026-12-06"],   // "年-月-日" の形式。空配列 [] なら非表示のまま
```

今日から見て一番近い「未来の」日付が自動で選ばれるので、1つの試験日が過ぎたら
リストの次の日付に**自動で切り替わります**（過ぎた日付を消す必要はありません）。
次回の試験日が決まったら、配列に追記して公開し直すだけでOKです。

残り日数が多いうちは王さまがのんびり励まし、直前になるとまおうがスパルタに焦らせてきます。
試験が終わったあとは（リストに次の日付がなければ）王さまが労いの言葉をかけ続けます。

## セキュリティについて（かならずお読みください）

このゲームは「静的サイト＋GAS Web App」というシンプルな構成のため、
**ゲーム側の仕組みだけで完全に外部からの不正アクセスを遮断することはできません**。
そのため、次の2段構えで個人情報の漏洩リスクを実質ゼロに抑える設計にしています。

1. **一覧の読み取り口（GET）を廃止**：生徒全員の名前・クラス・スコアをまとめて読み取れる
   APIは存在しません。ゲームができるのは「自分の成績を送信する」ことだけです。送信は
   一方通行なので、他人の情報が返ってくることもありません。
2. **成績一覧の閲覧はGoogle純正の共有機能に一任**：ランキングを見せたい場合は、
   スプレッドシートを「組織内限定・閲覧のみ」で共有します。これはGoogleが管理する
   確立された権限システムなので、学校の外の人には絶対に見えません。

補足として、送信（POST）用にAPIキー（合言葉）も設定していますが、これは
**「本気で調べようとする人」を完全には防げない簡易的な門番**です（ゲームは
静的サイトなのでソースコードは誰でも見られるため）。ただし送信APIは
「他人のデータを読み取る」ことができない設計なので、キーが仮に知られても
個人情報が漏れることはなく、最悪でも「いたずらでダミーの行が増える」程度の実害に留まります。
おかしなデータが登録された場合は、スプレッドシートの該当行を削除するだけで消せます。

念のため、**生徒には本名でなくニックネームでの登録を案内**しています（登録画面にも案内文を表示済み）。

## 問題の追加・編集のしかた

問題データは `game/js/questions_excel.js` と `game/js/questions_word.js` にあります。
プログラミングの知識がなくても、次の形をまねして書き足せます。

```javascript
{
  id: "XL2-9",                       // ← 他とかぶらないID。一度公開したら変えない！
  q: "問題文をここに書く",
  c: ["正解の選択肢", "はずれ1", "はずれ2", "はずれ3"],
  a: 0,                              // 正解の位置（0=1番目）。選択肢は自動でシャッフルされます
  exp: "解説をここに書く（まちがえたときに表示され、記憶定着を助けます）"
}
```

- **`id` は変更・使い回し禁止**です。生徒の復習記録（忘却曲線のデータ）がIDに紐づいています。
- 問題を消すときも、できれば残したまま入れ替えを。消すと生徒側の記録は無視されるだけで害はありません。

## よくある質問

**Q. 生徒のデータはどこに保存される？**
A. 生徒それぞれのブラウザ（localStorage）に保存されます。同じPC・同じブラウザで続きから遊べます。
ブラウザを変えたり履歴を全削除するとレベルはリセットされます（スプレッドシート側の記録は残ります）。

**Q. パスワードは？ なりすましが心配**
A. 授業用のカジュアルなゲームなのでパスワードはありません。名前とクラスだけで登録できます。
おかしな名前の登録があったら、スプレッドシートの該当行を削除すれば消えます。

**Q. 費用は本当に0円？**
A. はい。GitHub Pages / Netlify の無料枠、Google Apps Script・スプレッドシートの無料枠だけで動きます。
生徒数百人規模でも無料枠を超えることはまずありません。

**Q. スマホでも遊べる？**
A. 遊べます。画面はスマホ表示に対応しています。

**Q. ドラゴンクエスト等に似すぎていない？**
A. モンスターのデザイン・名前・称号・文言はすべてこのゲームのオリジナルです。
「レトロRPG風」という雰囲気（ジャンル）のみを踏襲し、既存作品の固有の名称・キャラクター・画像・音楽は一切使用していません。
